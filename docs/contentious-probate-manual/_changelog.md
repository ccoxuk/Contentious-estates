## Version Numbering Scheme

This manual uses semantic versioning:

**MAJOR.MINOR.PATCH**

- **MAJOR:** Significant structural changes or complete rewrites
- **MINOR:** New content, sections, or templates added
- **PATCH:** Corrections, updates, and minor improvements

**Examples:**
- `1.0.0` → Initial release
- `1.1.0` → Added new templates section
- `1.1.1` → Fixed typos and updated links
- `2.0.0` → Major restructure and rewrite
