## Versioning Policy

### Version Numbering

**Format:** MAJOR.MINOR.PATCH

- **MAJOR (X.0.0)**: Significant structural changes, complete rewrites, or fundamental approach changes
- **MINOR (0.X.0)**: New sections, templates, or substantial additions to existing content
- **PATCH (0.0.X)**: Minor corrections, updates, clarifications, or small improvements

### Release Schedule

**Major Versions:**
- As needed for significant changes
- Typically annual or bi-annual
- Extensive review and testing before release

**Minor Versions:**
- Quarterly or as needed
- Add new resources and guidance
- Moderate review process

**Patches:**
- As needed for corrections
- Quick turnaround
- Minimal review for non-controversial changes
