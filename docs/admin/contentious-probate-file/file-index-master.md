# File Index Master - Contentious Probate

<!-- This document provides a comprehensive file indexing system for contentious probate matters -->

## Purpose

To establish a standardized, comprehensive indexing system for contentious probate files, ensuring all documents are easily locatable and properly organized throughout the matter life cycle.

---

## File Index Structure

### Physical File Organization

**Recommended Division System (A-H):**

```
Section A: Client and Instructions
Section B: Parties and Opponents
Section C: Pleadings and Court Documents
Section D: Evidence
Section E: Correspondence
Section F: Financial and Costs
Section G: Research and Strategy
Section H: Miscellaneous
```

### Electronic File Organization

**DMS Folder Structure:**

```
/Matter-Reference/
  /01-Client-Instructions/
  /02-Parties-Opponents/
  /03-Pleadings/
  /04-Evidence/
    /04a-Witness-Statements/
    /04b-Expert-Reports/
    /04c-Documentary-Evidence/
  /05-Correspondence/
    /05a-Client/
    /05b-Opponent/
    /05c-Court/
    /05d-Third-Party/
  /06-Court-Documents/
  /07-Financial-Costs/
  /08-Research/
  /09-Bundles/
  /10-Archived/
```

## Master Document Register

### Template

**Date** | **Doc Type** | **Description** | **From/To** | **Section** | **Page/File**
---|---|---|---|---|---
[Date] | [Type] | [Description] | [Party] | [Section] | [Location]
2025-11-16 | Letter | Initial client care | To Client | A1.1 | p.1
2025-11-18 | Attendance | First meeting | Client | A2.1 | p.5
2025-11-20 | Letter | Letter before action | To Defendant | B2.2 | p.10

### How to Use

- Record EVERY document in/out
- Use consistent document type labels
- Include date received/sent
- Note section location
- Update promptly (ideally same day)
- Review monthly for accuracy

## Chronological Index

Maintain separate chronological index showing:

**Date** | **Event/Document** | **Reference**
---|---|---
[Date] | [Description] | [Section.subsection]

Example:
```
2025-11-16 | Client care letter sent | A1.1
2025-11-16 | Terms of business signed | A1.2
2025-11-18 | Initial client meeting | A2.1
2025-11-20 | Letter before action sent | B2.2
```

## Index Maintenance

### Responsibilities

- **Fee Earner:** Update index as documents added
- **Secretary/Support:** Assist with filing and indexing
- **Supervisor:** Review index during file reviews

### Review Schedule

- **Weekly:** Check index is current
- **Monthly:** Comprehensive review during file review
- **Before bundling:** Ensure complete and accurate

### Quality Checks

- [ ] All documents indexed
- [ ] Locations accurate
- [ ] No duplicates
- [ ] Consistent formatting
- [ ] Up to date

---

## Retrieval Procedures

### Finding Documents

1. Check master register by date/description
2. Identify section location
3. Go to physical section or electronic folder
4. Verify correct document

### If Document Missing

1. Check alternative sections
2. Check chronological index
3. Search electronic system
4. Check if with counsel/expert
5. Check if disclosed to opponent
6. Document search in file note
7. Request from source if necessary
