# File Naming Conventions - Contentious Probate

<!-- This document establishes standardized file naming conventions for contentious probate matters -->

## Purpose

To ensure consistent, logical, and descriptive file naming across all contentious probate matters, facilitating easy identification, retrieval, and management of documents.

---

## General Principles

### Must-Follow Rules

1. **Use hyphens** (-) not underscores (_) or spaces
2. **Use descriptive names** that explain content
3. **Include dates** in YYYY-MM-DD format
4. **Use version numbers** for drafts (v01, v02, etc.)
5. **Keep names concise** but informative
6. **Use consistent abbreviations**
7. **Avoid special characters** (!, @, #, $, %, etc.)
8. **Use title case** for readability

### Maximum Length

- Aim for 50-60 characters maximum
- Shorter is better if still descriptive

---

## Standard Format

### Primary Format

```
YYYY-MM-DD_Document-Type_Description_vNN.ext
```

**Components:**
- **YYYY-MM-DD:** Date (creation, sending, or relevant date)
- **Document-Type:** Category of document
- **Description:** Brief but clear description
- **vNN:** Version number (v01, v02, etc.) - omit for final versions
- **.ext:** File extension (.pdf, .docx, .xlsx, etc.)

### Examples

```
2025-11-16_Letter_To-Client-Re-Initial-Advice_v01.docx
2025-11-18_Attendance-Note_Client-Meeting.pdf
2025-11-20_Letter_To-Defendant-LBA.pdf
2025-11-25_Witness-Statement_J-Smith_v02.docx
2025-12-01_Claim-Form_Main-Claim.pdf
2025-12-05_Expert-Report_Dr-Jones-Capacity_FINAL.pdf
```

**Code** | **Meaning** | **Example**

**Code** | **Meaning** | **Example**

## Matter and Client References

### Matter Number in Filename (Optional but Recommended)

For large firms or when emailing externally:

```
CP-2025-1234_YYYY-MM-DD_Document-Type_Description.ext
```

Example:
```
CP-2025-1234_2025-11-20_LBA_To-Defendant.pdf
```

### Client Names (When Appropriate)

Use initials or abbreviated names:

```
2025-11-16_Letter_Smith-Estate_To-Executor.pdf
```

---

## Version Control

### Draft Versions

Use sequential version numbers:

```
v01, v02, v03, v04, ..., v10, v11, etc.
```

Examples:
```
2025-11-16_Witness-Statement_Client_v01.docx (first draft)
2025-11-18_Witness-Statement_Client_v02.docx (revised)
2025-11-20_Witness-Statement_Client_v03.docx (further revisions)
2025-11-22_Witness-Statement_Client_FINAL.docx (final version)
```

### Final Versions

Options for final versions:
- Omit version number entirely
- Use `_FINAL` suffix
- Use `_v00` (some firms' convention)

**Recommended:** Use `_FINAL` for clarity

```
2025-11-22_Particulars-of-Claim_FINAL.pdf
2025-12-01_Witness-Statement_Client_FINAL.pdf
```

### Superseded Versions

Move superseded drafts to "Archive" or "Drafts" subfolder

Alternatively, add "SUPERSEDED" or "ARCHIVE":
```
2025-11-16_Witness-Statement_Client_v01_SUPERSEDED.docx
```

## Special Document Types

### Bundles

```
YYYY-MM-DD_Bundle-Type_Description.pdf
```

Examples:
```
2026-02-15_Trial-Bundle_Volume-1.pdf
2026-02-15_Trial-Bundle_Volume-2.pdf
2026-01-15_Disclosure-Bundle.pdf
2026-02-10_Authorities-Bundle.pdf
```

### Bundles with Indices

```
2026-02-15_Trial-Bundle_Index.pdf
2026-02-15_Trial-Bundle_Volume-1.pdf
```

### Exhibits

Use witness initials and exhibit number:

```
YYYY-MM-DD_Exhibit_Initials-Number_Description.pdf
```

Examples:
```
2026-01-20_Exhibit_JS-1_Medical-Records.pdf
2026-01-20_Exhibit_JS-2_Correspondence.pdf
2026-01-20_Exhibit_AB-1_Will-File.pdf
```

### Amended Documents

```
YYYY-MM-DD_Document-Type_Description_AMENDED.pdf
```

Example:
```
2026-03-01_Particulars-of-Claim_AMENDED.pdf
```

## Correspondence Naming

### Letters to Clients

```
YYYY-MM-DD_Letter_To-Client_Subject.pdf
```

Examples:
```
2025-11-16_Letter_To-Client_Care-Letter.pdf
2025-11-20_Letter_To-Client_Advice-on-Merits.pdf
2025-12-01_Letter_To-Client_Update-Proceedings-Issued.pdf
```

### Letters from Clients

```
YYYY-MM-DD_Letter_From-Client_Subject.pdf
```

### Letters to/from Opponent

```
YYYY-MM-DD_Letter_To-Opponent_Subject.pdf
2025-11-20_Letter_From-Opponent_Response-to-LBA.pdf
```

### Court Correspondence

```
YYYY-MM-DD_Letter_To-Court_Subject.pdf
2025-12-05_Letter_From-Court_Hearing-Notice.pdf
```

---

## Privilege Marking

### Privileged Documents

Add "PRIVILEGED" to filename:

```
YYYY-MM-DD_Document-Type_Description_PRIVILEGED.ext
```

Examples:
```
2025-11-22_Advice_Counsel-Opinion_PRIVILEGED.pdf
2025-11-25_Email_To-Expert-Re-Instructions_PRIVILEGED.pdf
```

### Without Prejudice

Mark clearly:

```
YYYY-MM-DD_Without-Prejudice_Description.pdf
```

Example:
```
2025-12-10_Without-Prejudice_Settlement-Proposal.pdf
```

---

## Confidential Documents

Add "CONFIDENTIAL":

```
YYYY-MM-DD_Document-Type_Description_CONFIDENTIAL.ext
```

---

## Date Conventions

### Which Date to Use?

**General rule:** Use the most relevant date

**For outgoing documents:** Date created/sent
```
2025-11-20_Letter_To-Client_Advice.pdf
```

**For incoming documents:** Date received
```
2025-11-22_Letter_From-Opponent_Response.pdf
```

**For court documents:** Date filed/issued
```
2025-12-01_Claim-Form_FILED.pdf
```

**For undated documents:** Use date received or date relevant to
```
2025-11-15_Medical-Records_2020-2023.pdf
```

## Party Names in Filenames

### Use Short Forms

**Instead of:** `Particulars-of-Claim-Jane-Elizabeth-Smith-v-John-Robert-Jones`

**Use:** `Particulars_Smith-v-Jones`

Or use matter reference:
`CP-2025-1234_Particulars`

## Common Abbreviations

**Abbreviation** | **Meaning**
---|---
LBA | Letter Before Action
WS | Witness Statement
PoC | Particulars of Claim
CMC | Case Management Conference
PTR | Pre-Trial Review
AoS | Acknowledgment of Service
App | Application
Skel | Skeleton Argument

Use sparingly and only if clear in context.

---

## File Extensions

### Use Standard Extensions

**Document Type** | **Extension**
---|---
Word documents | .docx
PDF | .pdf
Excel spreadsheets | .xlsx
Images | .jpg or .png
Emails (saved) | .msg or .pdf

### Converting Formats

- **Working drafts:** Keep as .docx
- **Final versions/filing:** Convert to .pdf
- **Court filing:** Usually .pdf required

## Non-Compliant Examples (To Avoid)

**Bad:** `letter to client.doc`  
**Good:** `2025-11-16_Letter_To-Client_Initial-Advice.pdf`

**Bad:** `ws v1.docx`  
**Good:** `2026-01-20_Witness-Statement_Client_v01.docx`

**Bad:** `Final Particulars of claim FINAL (2).pdf`  
**Good:** `2025-12-01_Particulars-of-Claim_FINAL.pdf`

**Bad:** `copy of email from John.msg`  
**Good:** `2025-11-25_Email_From-Opponent_Re-Disclosure.pdf`

## Transitioning to New Conventions

### For Existing Matters

- Adopt conventions for all new documents
- Optionally rename key documents in existing files
- Update index to reflect new names

### Training

- All staff trained on conventions
- Examples provided
- Regular audits to ensure compliance
